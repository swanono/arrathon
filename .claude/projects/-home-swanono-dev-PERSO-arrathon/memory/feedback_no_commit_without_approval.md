---
name: No commit without approval
description: Never commit without explicit user approval first
type: feedback
---

Never create a git commit without explicitly asking the user for approval first and waiting for their go-ahead.

**Why:** User has been frustrated multiple times by commits being made without their consent. They want full control over what gets committed and when.

**How to apply:** Always show what would be committed and ask "Tu veux que je commit ?" before running any `git commit` command. No exceptions.
